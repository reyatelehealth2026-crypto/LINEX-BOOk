# คู่มือการใช้งาน LineBook แบบละเอียด

อ้างอิงสไตล์: คู่มือแบบ Notion Product Manual ผสม SOP ใช้งานหน้างาน เน้นอ่านง่าย ทำตามได้จริง

คู่มือนี้แบ่งเป็น 2 ฝั่ง:
- **ฝั่งแอดมิน / เจ้าของร้าน** สำหรับตั้งค่าระบบและจัดการคิว
- **ฝั่งลูกค้า** สำหรับจองคิวผ่าน LINE และ LIFF Mini App

> เหมาะกับร้านตัดผม ร้านเสริมสวย ร้านทำเล็บ และร้านบริการที่รับจองคิวเป็นช่วงเวลา

---

# 1) ภาพรวมระบบ

LineBook คือระบบจองคิวผ่าน LINE โดยมี 2 ประสบการณ์หลัก

1. **LINE Chat / Flex**
   - ลูกค้าคุยกับ LINE OA ของร้าน
   - กดปุ่มจองจาก Rich Menu หรือพิมพ์ข้อความเพื่อเริ่มจอง
   - บาง flow ใช้ Flex message ช่วยเลือกบริการ วัน และเวลา

2. **LIFF Mini App**
   - เป็นหน้าเว็บที่เปิดภายใน LINE
   - ใช้สำหรับจองคิว ดูคิวของฉัน แก้โปรไฟล์ เปลี่ยนเวลา รีวิว และจองซ้ำ

ฝั่งร้านจะมี **Admin Panel** สำหรับ:
- ดูคิวรายวัน
- เปลี่ยนสถานะคิว
- ดู dashboard
- จัดการบริการ / พนักงาน / เวลาทำการ
- ดูลูกค้าและประวัติ
- จัดการ message templates
- ตรวจ healthcheck และ setup ความพร้อมของระบบ

# 2) คู่มือฝั่งแอดมิน / เจ้าของร้าน

## 2.1 เข้าใช้งานหน้าแอดมิน

1. เปิด URL หน้าแอดมินของระบบ
2. ระบบจะขึ้นหน้า **Admin Login**
3. ใส่รหัสผ่านที่ตั้งไว้ใน `ADMIN_PASSWORD`
4. เมื่อเข้าได้แล้ว จะเห็นเมนูหลักด้านบน เช่น
   - แดชบอร์ด
   - คิววันนี้
   - ตั้งค่าร้าน
   - ตรวจสอบระบบ
   - บริการ
   - พนักงาน
   - เวลาทำการ
   - ลูกค้า
   - เทมเพลต

> หมายเหตุ: รหัสผ่านถูกเก็บใน session ของ browser เครื่องนั้น ถ้ากด “ออกจากระบบ” จะต้องใส่ใหม่

---

## 2.2 เริ่มต้นใช้งานครั้งแรก

แนะนำให้เข้าเมนู **ตั้งค่าร้าน** ก่อนเสมอ

### เมนู: `/admin/setup`
หน้านี้ออกแบบเป็น setup wizard มี progress bar และ checklist ให้ทำทีละข้อ

### สิ่งที่ต้องทำ

#### ขั้นที่ 1: ตั้งค่า Supabase
ต้องมีค่าเหล่านี้ใน `.env`
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`

หน้าตั้งค่าจะตรวจให้อัตโนมัติว่าใส่ครบหรือยัง

#### ขั้นที่ 2: ตั้งค่า LINE Official Account
ต้องมีค่า
- `LINE_CHANNEL_ACCESS_TOKEN`
- `LINE_CHANNEL_SECRET`

ใช้สำหรับให้ระบบตอบลูกค้าและส่งข้อความยืนยัน/แจ้งเตือน

#### ขั้นที่ 3: สร้าง LIFF App
ต้องมีค่า
- `NEXT_PUBLIC_LIFF_ID`

ใช้สำหรับเปิด Mini App ใน LINE เช่นหน้าจองคิว หน้าโปรไฟล์ และหน้าคิวของฉัน

#### ขั้นที่ 4: ตั้งค่า Webhook
ใน LINE Developers Console ให้ใส่ Webhook URL ของระบบไปที่
- `https://<your-domain>/api/line/webhook`

และเปิด **Use webhook**

#### ขั้นที่ 5: ตั้งรหัสผ่านแอดมิน
ต้องมีค่า
- `ADMIN_PASSWORD`

#### ขั้นที่ 6: ใส่ข้อมูลร้านพื้นฐาน
อย่างน้อยควรมี
- บริการ 1 รายการขึ้นไป
- พนักงาน 1 คนขึ้นไป
- เวลาทำการอย่างน้อย 1 ช่วง

เมื่อครบทุกขั้น หน้า setup จะขึ้นว่า **ร้านพร้อมใช้งานแล้ว**

---

## 2.3 ตรวจสอบระบบ

### เมนู: `/admin/healthcheck`
ใช้สำหรับตรวจว่าอะไรพร้อมหรือยังไม่พร้อม

### ระบบจะเช็คให้ เช่น
- LINE token
- LINE secret
- LIFF ID
- LINE API ติดต่อได้หรือไม่
- Supabase URL / keys
- เชื่อมฐานข้อมูลได้หรือไม่
- ตั้ง ADMIN_PASSWORD แล้วหรือยัง
- มีข้อมูลร้าน / บริการ / พนักงานแล้วหรือยัง

### วิธีอ่านผล
- **ผ่าน** = พร้อมใช้งาน
- **Warn** = ยังไม่ครบ แต่ยังไม่ถึงขั้นพัง
- **Fail** = ต้องแก้ก่อนระบบจะทำงานสมบูรณ์

ถ้าผ่านทั้งหมด หน้าจอนี้จะโชว์ URL สำคัญให้คัดลอกได้ เช่น
- Webhook URL
- LIFF URL
- App URL

เหมาะมากเวลาจะส่งให้ทีม deploy หรือ owner คัดลอกไปตั้งค่าเอง

## 2.4 จัดการคิววันนี้

### เมนู: `/admin`
หน้านี้คือหน้าใช้งานประจำวันของร้าน

### ทำอะไรได้บ้าง
- ดูคิวของวันนั้นทั้งหมด
- เปลี่ยนวันที่เพื่อดูคิววันอื่น
- ดูจำนวนคิวแต่ละสถานะ
- ดูข้อมูลลูกค้าในแต่ละคิว
- เปลี่ยนสถานะคิวได้ทันที

### ข้อมูลที่เห็นในการ์ดคิว
- เวลาเริ่ม
- ระยะเวลาบริการ
- ชื่อลูกค้า
- เบอร์โทร (ถ้ามี)
- บริการ
- ช่าง
- ราคา
- จำนวนครั้งที่เคยมา
- แต้มสะสม
- หมายเหตุจากลูกค้า (ถ้ามี)

### ปุ่มสถานะที่ใช้ได้
- **ยืนยัน**: ใช้เมื่อรับคิวแล้ว
- **เสร็จ**: ใช้เมื่อให้บริการเสร็จ
- **ไม่มา**: ใช้เมื่อลูกค้า no-show
- **ยกเลิก**: ใช้เมื่อลูกค้ายกเลิกหรือร้านยกเลิก

### ผลของการเปลี่ยนสถานะ
- เมื่อเปลี่ยนเป็น **completed** ระบบสามารถนำไปใช้คำนวณ
  - รายได้
  - visit count
  - points
  - review eligibility
  - quick rebook history

> หน้าแอดมินนี้รองรับ realtime เมื่อมี booking ใหม่ รายการจะรีเฟรชตามข้อมูลในระบบ

---

## 2.5 ดูแดชบอร์ดเจ้าของร้าน

### เมนู: `/admin/dashboard`
ใช้ดูภาพรวมธุรกิจรายวันและรายสัปดาห์

### มีข้อมูลอะไรบ้าง

#### ภาพรวมร้าน
- ลูกค้าทั้งหมด
- บริการที่เปิดอยู่
- จำนวนพนักงานที่ active

#### สรุปวันนี้
- คิวทั้งหมด
- รอยืนยัน
- ยืนยันแล้ว
- เสร็จสิ้น
- ยกเลิก
- ไม่มา

#### รายได้
- รายได้วันนี้จากคิวที่เสร็จสิ้น
- รายได้โดยประมาณจากคิว pending + confirmed
- รายได้สัปดาห์นี้

#### Breakdown
- แยกตามบริการ
- แยกตามช่าง
- คิวที่กำลังจะมาถึง

### ใช้งานเมื่อไร
- ตอนเช้าเพื่อดูภาพรวมวันนี้
- ระหว่างวันเพื่อเช็คงานแน่น/หลวม
- ตอนปิดร้านเพื่อดูยอดและสถานะคิว

---

## 2.6 จัดการบริการ

### เมนู: `/admin/services`
ใช้เพิ่ม/แก้ไข/ปิดใช้งานบริการของร้าน

### ฟิลด์ที่ตั้งค่าได้
- ชื่อบริการ
- ชื่อภาษาอังกฤษ
- ระยะเวลา (นาที)
- ราคา
- ลำดับแสดงผล
- เปิดใช้งาน / ปิดใช้งาน
- รายละเอียดบริการ

### วิธีเพิ่มบริการ
1. กด **เพิ่มบริการ**
2. กรอกชื่อ เช่น “ตัดผมชาย”
3. ใส่ระยะเวลา เช่น 45 นาที
4. ใส่ราคา เช่น 250 บาท
5. กดบันทึก

### วิธีปิดบริการชั่วคราว
- กด toggle จากเปิดเป็นปิด
- ลูกค้าจะไม่เห็นบริการนั้นใน flow จอง

### คำแนะนำ
- ตั้งระยะเวลาให้ใกล้ของจริงที่สุด เพราะระบบคำนวณ slot จากตัวนี้
- ใช้ sort order เพื่อเรียงบริการขายดีไว้บนสุด

---

## 2.7 จัดการพนักงาน

### เมนู: `/admin/staff`
ใช้จัดการรายชื่อช่างหรือพนักงานรับงาน

### ฟิลด์ที่ตั้งค่าได้
- ชื่อจริง
- ชื่อเล่น
- รูปโปรไฟล์ (URL)
- ข้อมูลเพิ่มเติม / ความเชี่ยวชาญ
- ลำดับแสดงผล
- เปิดใช้งาน / ปิดใช้งาน

### วิธีใช้จริง
- ใส่ชื่อเล่นเพื่อให้ลูกค้าเห็นชื่อที่จำง่าย เช่น “พี่โอ๋”
- ใส่ bio เช่น “ชำนาญทำสี / สระไดร์ / ตัดผมชาย”
- ถ้าพนักงานลาหยุดยาว ให้ปิด active ชั่วคราว

## 2.8 จัดการเวลาทำการ

### เมนู: `/admin/working-hours`
ใช้กำหนดเวลาเปิด-ปิด ทั้งระดับร้านและระดับช่าง

### รองรับ 2 แบบ
1. **ทั้งร้าน**
   - ใช้เป็นเวลามาตรฐานของร้าน
2. **รายช่าง**
   - ใช้ override เวลาเฉพาะคน

### วิธีเพิ่มเวลาทำการ
1. กด **เพิ่มเวลาทำการ**
2. เลือกวัน
3. เลือกช่าง หรือปล่อยว่างเพื่อใช้เป็น “ทั้งร้าน”
4. ใส่เวลาเปิดและเวลาปิด
5. กดบันทึก

### วิธีแก้เวลาทำการเดิม
- แก้เวลาในรายการเดิม
- ถ้ามีการเปลี่ยนจริง กด **บันทึก**

### วิธีลบ
- กดไอคอนถังขยะ

### ข้อควรระวัง
- ถ้าตั้งเวลาไม่ครบ ระบบอาจหา slot ให้ลูกค้าไม่ได้
- ถ้ามีหลายบรรทัดซ้ำวันเดียวกัน ควรตรวจให้แน่ใจว่าเป็นช่วงที่ตั้งใจจริง

## 2.10 ดูโปรไฟล์ลูกค้าแบบละเอียด

### เมนู: `/admin/customers/[id]`
ใช้ดู timeline ของลูกค้าแต่ละคน

### ข้อมูลที่เห็น

- แต้มสะสม
- สถานะลงทะเบียน
- จำนวนครั้งที่จองทั้งหมด
- จำนวนครั้งที่เสร็จสิ้น
- จำนวนครั้งที่ยกเลิก / ไม่มา

### Booking timeline

## 2.11 จัดการเทมเพลตข้อความ

### เมนู: `/admin/templates`
ใช้จัดการข้อความมาตรฐานของร้าน

### ใช้ทำอะไรได้
- สร้าง template สำหรับเตือนนัด
- ทำข้อความโปรโมชั่น
- ทำ follow-up หลังใช้บริการ
- preview ข้อความก่อนใช้งานจริง

### หมวดหมู่ที่มี
- Reminder
- Promo
- Follow-up
- Custom

### ตัวแปรที่รองรับ
- `{{customer_name}}`
- `{{service_name}}`
- `{{date}}`
- `{{time}}`
- `{{shop_name}}`
- `{{staff_name}}`

### วิธีสร้างเทมเพลต
1. กด **สร้างเทมเพลต**
2. ใส่ชื่อภายใน เช่น “เตือนนัด 1 ชั่วโมง”
3. เลือกหมวดหมู่
4. ใส่หัวข้อ (ถ้าต้องการ)
5. ใส่ข้อความหลัก
6. แทรกตัวแปรที่ต้องการ
7. ตั้ง sort order
8. กดบันทึก

### ตัวอย่าง
```text
สวัสดีค่ะ {{customer_name}}
ขอเตือนว่าคุณมีนัด {{service_name}} วันที่ {{date}} เวลา {{time}}
หากต้องการเลื่อนนัด สามารถทักแชทร้านได้เลยค่ะ
```

### สิ่งที่ระบบทำได้ตอนนี้
- CRUD template
- เปิด/ปิดการใช้งาน
- preview ข้อความ

### สิ่งที่ยังเป็น groundwork
- ยังไม่มี bulk campaign sender เต็มรูปแบบ
- ยังไม่มี audience segmentation เต็มระบบ

## 2.12 การจัดการผ่าน LINE Chat (สำหรับแอดมินที่ระบบอนุญาต)

ถ้าระบบตั้ง LINE user ของเจ้าของร้านเป็น admin แล้ว จะมีคำสั่งแชทบางส่วนที่ใช้ได้จากใน LINE โดยตรง

### ดูคิว
- `คิว`
- `คิววันนี้`
- `คิวพรุ่งนี้`
- `คิว 20/4`

### ดูยอด
- `ยอด`
- `ยอดวันนี้`
- `รายได้`

### เปลี่ยนสถานะคิว
- `ยืนยัน #5`
- `เสร็จ #5`
- `ยกเลิก #5`
- `ไม่มา #5`

### เพิ่มข้อมูลพื้นฐานเร็วๆ
- `เพิ่มบริการ ทำสี 1200 บาท 120 นาที`
- `เพิ่มช่าง พี่มิ้น`

> คำสั่งชุดนี้เหมาะกับงานเร็วระหว่างวัน แต่ถ้าต้องตั้งค่าละเอียด แนะนำใช้หน้าเว็บ admin จะชัดกว่า

# 3) คู่มือฝั่งลูกค้า

ลูกค้าใช้งานผ่าน 2 ทางหลัก
- คุยกับ LINE OA ของร้าน
- เปิด LIFF Mini App ภายใน LINE

---

## 3.1 เริ่มต้นจาก LINE OA

เมื่อลูกค้าแอด LINE OA ร้านแล้ว จะได้รับ welcome message และสามารถเริ่มใช้งานได้จาก
- Rich Menu
- ปุ่มใน Flex message
- หรือพิมพ์ข้อความเอง

### ตัวอย่างคำที่ลูกค้าพิมพ์ได้
- `จอง`
- `จองคิว`
- `คิว`
- `แต้ม`
- `โปรไฟล์`
- `บริการ`
- `ราคา`

### ตัวอย่างการจองแบบภาษาธรรมชาติ
- `จองตัดผมพรุ่งนี้บ่ายสอง`
- `ขอจองทำสีวันศุกร์ 11 โมง`
- `จองตัดผมกับพี่โอ๋พรุ่งนี้ 15:00`

ถ้าระบบจับ intent ได้ จะช่วยพาไปต่อใน flow ที่เหมาะสม

---

## 3.2 จองคิวแบบทีละขั้นผ่าน LIFF

### หน้า: `/liff/booking`
Flow หลักมี 4 ขั้น

#### ขั้นที่ 1: เลือกบริการ
ลูกค้าจะเห็นรายการบริการ เช่น
- ตัดผมชาย
- ตัดผม + สระ
- ทำสีผม

#### ขั้นที่ 2: เลือกช่าง
เลือกได้ 2 แบบ
- ช่างคนไหนก็ได้
- เลือกช่างเฉพาะคน

#### ขั้นที่ 3: เลือกวันและเวลา
- เลือกวันภายในช่วงที่ระบบเปิดให้จอง
- ระบบจะแสดงเฉพาะ slot ที่ว่างจริง
- ถ้าไม่มีคิวว่าง ระบบจะมีปุ่ม **แจ้งเตือนเมื่อมีคิวว่าง** เพื่อเข้ารอคิว

#### ขั้นที่ 4: ยืนยัน
ลูกค้าจะเห็นสรุป
- บริการ
- เวลา
- ราคา
- ช่าง
- หมายเหตุ

จากนั้นกด **ยืนยัน**

### หลังจองสำเร็จ
- ระบบสร้าง booking สถานะ confirmed
- ลูกค้าได้รับข้อความยืนยันผ่าน LINE
- คิวจะไปอยู่ในหน้า “คิวของฉัน”

---

## 3.3 เข้ารอคิว (Waitlist)

ถ้าวัน/เวลาที่ต้องการเต็ม ลูกค้าสามารถกด
- **แจ้งเตือนเมื่อมีคิวว่าง**

### สิ่งที่เกิดขึ้น
- ระบบบันทึก waitlist entry
- ถ้ามี slot ว่างจากการยกเลิกหรือเลื่อนเวลา ระบบมีพื้นฐานสำหรับแจ้งลูกค้า
- ลูกค้าจะเห็นข้อความยืนยันว่าเข้ารอคิวแล้ว

### เหมาะกับกรณี
- ต้องการวันเดิม แต่คิวเต็ม
- ลูกค้า flexible และรอได้

---

## 3.4 ดูคิวของฉัน

### หน้า: `/liff/my-bookings`
ใช้ดู booking ทั้งหมดของลูกค้า

### ในแต่ละรายการจะเห็น
- ชื่อบริการ
- วันเวลา
- ช่าง
- ราคา
- สถานะ

### ปุ่มที่อาจเจอ

#### ถ้าคิวยังไม่เสร็จ
- **เปลี่ยนเวลา**
- **ยกเลิก**

#### ถ้าคิวเสร็จแล้ว
- **จองอีกครั้ง**
- **รีวิว**

---

## 3.5 เปลี่ยนเวลา (Reschedule)

### หน้า: `/liff/reschedule`
ลูกค้าสามารถเลื่อนคิวเองได้สำหรับคิวที่ยัง active เช่น pending หรือ confirmed

### Flow
1. กดปุ่ม **เปลี่ยนเวลา** จากหน้า “คิวของฉัน”
2. ระบบแสดงข้อมูลคิวเดิม
3. เลือกวันใหม่
4. เลือกเวลาว่างใหม่
5. กดยืนยัน

### หลังเปลี่ยนเวลา
- booking เดิมถูกอัปเดตเวลาใหม่
- ลูกค้าได้รับข้อความยืนยันการเปลี่ยนเวลา
- คิวเดิมในรายการถูกอัปเดต

> ถ้าเวลาที่เลือกมีคนจองไปก่อน ระบบจะแจ้งว่าเวลานั้นไม่ว่าง

---

## 3.6 ยกเลิกคิว

จากหน้า “คิวของฉัน” ลูกค้ากด **ยกเลิก** ได้ในคิวที่ยังไม่จบ

### หลังยกเลิก
- สถานะ booking จะเปลี่ยนเป็น `cancelled`
- ระบบสามารถนำ slot นั้นไปใช้แจ้ง waitlist ต่อได้

---

## 3.7 จองอีกครั้ง (Quick Rebook)

สำหรับคิวที่เสร็จแล้ว ลูกค้าจะเห็นปุ่ม **จองอีกครั้ง**

### วิธีทำงาน
- ระบบพาลูกค้ากลับไปหน้าจอง
- prefill บริการเดิม
- prefill ช่างเดิม (ถ้ามี)
- ลูกค้าเลือกวัน/เวลาใหม่ต่อได้เลย

### เหมาะกับลูกค้าประจำ
เช่น ลูกค้าที่ตัดผมกับช่างคนเดิมทุกเดือน

---

## 3.8 รีวิวหลังใช้บริการ

### หน้า: `/liff/review`
สำหรับคิวที่เสร็จแล้ว ลูกค้าสามารถให้คะแนนได้

### ทำอะไรได้บ้าง
- ให้ดาว 1–5 ดาว
- เขียนคอมเมนต์เพิ่มเติม
- ส่งรีวิวได้ 1 ครั้งต่อ booking

### หลังส่งรีวิว
- ระบบบันทึกลงฐานข้อมูล
- ลูกค้าจะเห็นหน้าขอบคุณ
- มีปุ่มให้ไปจองอีกครั้งได้

---

## 3.9 โปรไฟล์ลูกค้า / ลงทะเบียน

### หน้า: `/liff/profile`
ใช้สำหรับกรอกข้อมูลลูกค้า

### ข้อมูลที่กรอกได้
- ชื่อจริง
- เบอร์โทร
- วันเกิด

### ประโยชน์ของการลงทะเบียน
- ระบบจดจำข้อมูลลูกค้าได้ดีขึ้น
- ร้านติดต่อกลับง่ายขึ้น
- ลูกค้าเห็นแต้มสะสม
- เห็นจำนวนครั้งที่เคยใช้บริการ
- ครั้งแรกอาจได้รับแต้มต้อนรับ

### สิ่งที่เห็นหลังลงทะเบียน
- แต้มสะสม
- จำนวนครั้งที่มาใช้บริการ

---

## 3.10 ดูบริการและราคา

ลูกค้าดูรายการบริการได้จาก

ระบบจะพาไปดูรายการบริการพร้อมราคา

# 5) ปัญหาที่พบบ่อย

## ลูกค้ากดจองแล้วไม่เห็นเวลา
ตรวจที่
- มีบริการ active หรือยัง
- มีพนักงาน active หรือยัง
- ตั้ง working hours แล้วหรือยัง
- วันนั้นมี slot ว่างหรือไม่

## LINE ไม่ตอบลูกค้า
ตรวจที่
- Webhook URL ถูกหรือไม่
- เปิด Use webhook แล้วหรือยัง
- LINE token / secret ถูกหรือไม่
- OA อยู่โหมดให้คนตอบอยู่หรือไม่

## หน้าแอดมินเข้าไม่ได้
ตรวจที่
- `ADMIN_PASSWORD` ถูกหรือไม่
- deploy env ใหม่แล้วหรือยัง
- browser session เก่าค้างหรือไม่

## ลูกค้าเปลี่ยนเวลาไม่ได้
ตรวจที่
- booking นั้นยังเป็น pending / confirmed หรือไม่
- เวลาใหม่ชนกับคิวอื่นหรือไม่

## รีวิวไม่ได้
ตรวจที่
- booking ต้องเป็น `completed`
- booking เดิมต้องยังไม่เคยรีวิวมาก่อน

- review automation หลัง completed แบบเต็มระบบ

# 7) คำแนะนำการใช้งานให้ลื่นที่สุด

## สำหรับร้าน
- ตั้งชื่อบริการให้ชัดและสั้น
- ตั้งเวลาบริการตามจริง
- ปิด inactive บริการ/ช่างที่ไม่พร้อมรับงาน

- ใช้ customer timeline เพื่อดูว่าลูกค้าคนไหนกลับมาซ้ำ

## สำหรับลูกค้า

- ถ้าคิวเต็ม ให้กดเข้ารอคิวแทนการทักถามหลายรอบ
- ใช้ปุ่ม “จองอีกครั้ง” เมื่อต้องการจองซ้ำแบบเร็ว
